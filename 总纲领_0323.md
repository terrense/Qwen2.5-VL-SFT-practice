## 一、实验目标与架构

### 1.1 目标

在 **Qwen2.5-VL-32B-Instruct** 基座模型上，使用 **LoRA** 进行 1 个 epoch 的监督微调（SFT），面向**面部症状识别**任务。输出为结构化 JSON，包含四个字段：`pain_status`、`facial_asymmetry`、`gender`、`levine_sign`。

### 1.2 整体流程

```
基座模型 → [可选] 基座评测 → LoRA 训练 → LoRA 评测 → 单图推理
```

| 阶段 | 脚本 | 作用 |
|------|------|------|
| 基座评测 | `eval_base_0323.py` | 在验证集上评测未微调基座，作 baseline |
| LoRA 训练 | `train_lora_full_0323.py` | 1 epoch LoRA 微调 |
| LoRA 评测 | `eval_lora_0323.py` | 在验证集上评测微调后模型 |
| 单图推理 | `inference_lora_0323.py` | 对单张图片做推理演示 |

## 二、环境配置（必读）

### 2.1 基础要求

- Python ≥ 3.8
- CUDA（推荐 11.8+）
- 显存：32B 模型 + LoRA 训练约需 40GB+（bf16 + gradient_checkpointing）

### 2.2 推荐：Conda 创建独立环境

```bash
# 创建环境
conda create -n qwen25vl_sft_0323 python=3.10 -y
conda activate qwen25vl_sft_0323

# 安装 PyTorch（按 CUDA 版本选择，示例为 CUDA 11.8）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
```

### 2.3 安装项目依赖

```bash
cd /data/sx_files/qwen25vl_sft_chain_0323
pip install -r requirements_0323.txt
```

### 2.4 关键依赖说明：qwen_vl_utils

代码中使用 `from qwen_vl_utils import process_vision_info`。若 IDE 出现**波浪线/红色提示**，多为该包未安装或当前解释器未选中正确环境。

**安装方式：**

```bash
pip install qwen-vl-utils
```

- PyPI 包名：`qwen-vl-utils`
- 导入名：`qwen_vl_utils`（注意下划线）

**验证安装：**

```bash
python -c "from qwen_vl_utils import process_vision_info; print('OK')"
```

若输出 `OK` 则安装成功。若仍报错，可尝试：

```bash
pip install qwen-vl-utils -U
```

### 2.5 环境验证脚本

在项目根目录执行：

## 五、常见问题

### Q1：`ModuleNotFoundError: No module named 'qwen_vl_utils'`

- 执行：`pip install qwen-vl-utils`
- 确认当前 Python 环境与运行脚本时一致（如 `conda activate qwen25vl_sft_0323`）

### Q2：IDE 中 `qwen_vl_utils` 有波浪线但运行正常

- 多因 IDE 使用的解释器与实际运行环境不同。在 IDE 中切换解释器到项目所用 conda/venv 即可。

### Q3：LoRA 评测报错 `LoRA checkpoint not found`

- 需先完成训练：`./run_train_0323.sh`
- 确认 `runs/qwen25vl32b_full_1epoch_0323` 目录存在且含 `adapter_config.json` 等文件

### Q4：显存不足

- 已启用 `gradient_checkpointing` 与 `bf16`。若仍不足，可尝试减小 `gradient_accumulation_steps` 或使用多卡。
