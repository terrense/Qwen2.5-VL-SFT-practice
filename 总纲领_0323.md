# Qwen2.5-VL-32B 全链路总纲领 (0323)

本文档是当前 `_0323` 目录的总设计说明，目标不是只描述 SFT，而是把训练、后处理、推理部署统一成一条能实际执行的完整链路。

## 一、目标与架构

### 1.1 任务目标

在 `Qwen2.5-VL-32B-Instruct` 上完成面部症状识别的结构化输出，输出字段固定为：

- `pain_status`
- `facial_asymmetry`
- `gender`
- `levine_sign`

### 1.2 总体架构

```text
数据与标签
  -> 基座评测
  -> LoRA SFT
  -> LoRA评测
  -> 单图推理验证
  -> merge 成完整模型
  -> 2:4 结构化剪枝
  -> 剪枝/merged 模型评测
  -> 量化导出
  -> TensorRT-LLM step1 导出
  -> TensorRT-LLM step2 构建 engine
  -> OpenAI-compatible API 部署
```

这里把链路拆成三层：

- 实验层：`baseline -> train -> eval -> inference`
- 后处理层：`merge -> prune -> pruned eval -> quantize`
- 部署层：`TensorRT export/build -> API service`

这次目录设计的核心原则是：

- 根目录只保留训练、评测、推理和总配置。
- `post_sft/` 专门放 SFT 之后的模型处理脚本。
- `quantization/`、`trt/`、`deploy/` 分层放部署相关代码。
- `scripts/` 统一放启动脚本，避免继续把 `run_*.sh` 全堆在根目录。

这意味着后续你要迁移路径时，优先改这里，而不是逐个改脚本。

## 四、执行顺序

### 4.1 基础实验顺序

```bash
cd /data/sx_files/qwen25vl_sft_chain_0323
chmod +x run_*.sh scripts/*.sh
./run_eval_base_0323.sh
./run_train_0323.sh
./run_eval_lora_0323.sh
./run_inference_0323.sh
```

### 4.2 完整工程顺序

```bash
./scripts/run_merge_lora_0323.sh
./scripts/run_prune_0323.sh
./scripts/run_eval_pruned_0323.sh
./scripts/run_quantize_0323.sh
./scripts/run_export_onnx_0323.sh [TensorRT-LLM builder args...]
./scripts/run_build_engine_0323.sh --plan-path /path/to/build_plan_0323.json
./scripts/run_deploy_api_0323.sh --mode merged
```

推荐理解方式：

- 想做论文/实验闭环，只做到 LoRA eval 和 inference 即可。
- 想做部署闭环，至少继续做到 merge。
- 想做 TensorRT-LLM 高性能推理，再继续走 quantize 和 trt 两步构建。

## 五、每一层脚本职责

### 5.1 实验层

- `eval_base_0323.py`：建立基座 baseline。
- `train_lora_full_0323.py`：执行 1 epoch LoRA SFT。
- `eval_lora_0323.py`：验证微调收益。
- `inference_lora_0323.py`：快速做单图可用性检查。

### 5.2 后处理层

- `post_sft/merge_lora_0323.py`：把 LoRA adapter 合并进基座，导出独立完整模型。
- `post_sft/prune_2to4_0323.py`：对 merged 模型做 2:4 结构化剪枝。
- `post_sft/eval_pruned_0323.py`：重新评测 merged 或 pruned 模型，保证后处理没把指标打穿。

### 5.3 部署层

- `quantization/quantize_0323.py`：导出 `int4_awq` 或 `fp8` 量化结果。
- `trt/step1_export_0323.py`：导出 ONNX，并写 `build_plan_0323.json`。
- `trt/step2_build_0323.py`：读取计划文件，独立进程构建 TensorRT engine。
- `deploy/deploy_api_0323.py`：提供 OpenAI-compatible 多模态接口 `/v1/chat/completions`。

## 八、常见问题

### Q1：`Floating point exception` 又出现怎么办？

先确认不是环境回退了。当前已验证稳定组合是：

- `transformers==4.57.5`
- `torch==2.9.0+cu128`

如果环境被重新装过，优先检查这两个版本。

### Q2：`qwen_vl_utils` 报红线怎么办？

```bash
pip install qwen-vl-utils
python -c "from qwen_vl_utils import process_vision_info; print('OK')"
```

### Q3：为什么量化和 TensorRT 依赖没直接写进基础 requirements？

因为这部分不是普通 Python 纯包依赖，而是强依赖 CUDA / TensorRT / TensorRT-LLM 版本匹配，通常要单独建环境。
